❓ What does ? mean in the TTY column?

The ? means the process has no controlling terminal. This is the hallmark of a daemon process — it was detached from any terminal (via setsid()) and runs silently in the background. Examples: systemd, nginx, sshd.

Processes with a real terminal show entries like pts/0, tty1, etc.

🔢 Maximum PID & Wrap-around

$ cat /proc/sys/kernel/pid_max # usually 4194304 on 64-bit
  • Default on 32-bit systems: 32768
  • Default on 64-bit systems: 4,194,304 (4M)
  • PIDs 1–300 are reserved for kernel threads at boot
  • When the counter hits pid_max it wraps around back to 300 and skips any still-in-use PIDs
  • PID 1 is always systemd (or init) — the ancestor of all processes
$ sudo sysctl -w kernel.pid_max=65536 # change max PID

🔄 States of a Process

StateSymbol in psMeaning

Uninterruptible SleepDWaiting for kernel I/O (e.g. disk). Cannot be killed easily

ZombieZExited but parent hasn't called wait(). Entry still in process table
TracedtStopped under a debugger (ptrace)
⚠️ Zombie & Orphan

Zombie: Child exits before parent calls wait(). The entry stays in the process table until parent reaps it. Harmless in small numbers but accumulation wastes PIDs.

Orphan: Parent dies before child. The child is re-parented to init (PID 1), which periodically calls wait().

👻 Daemon Process

A daemon is a background service that starts at boot, runs without a terminal, and is generally a child of PID 1 (systemd). Examples: sshd, cron, nginx.

Daemonize Recipe (from examples/03_daemon.c)

// 1. fork() – parent exits, child continues
pid_t pid = fork();
if (pid > 0) exit(0);

// 2. setsid() – become session leader, detach terminal
setsid();

// 3. fork() again – can't accidentally reacquire a terminal
pid = fork();
if (pid > 0) exit(0);

// 4. chdir("/") – don't block unmounting
chdir("/");

// 5. umask(0)  – don't inherit parent's file permissions
umask(0);

// 6. Close all fds, redirect stdin/stdout/stderr → /dev/null
C
$ ps -e | grep nginx # ? in TTY column = daemon

🚀 System Boot Flow

  1. BIOS/UEFI – power-on self-test, finds bootable disk
  2. GRUB/bootloader – loads the kernel image (vmlinuz) from disk
  3. Kernel init – decompresses, sets up memory, mounts initial RAM disk (initrd)
  4. PID 1 – systemd – first user-space process; reads unit files, starts services
  5. getty / login – terminal login, then user shell
$ systemd-analyze blame # see what slowed boot
$ dmesg | head -30 # kernel ring buffer messages
$ pstree -p # visualize process hierarchy from PID 1

📝 Example

Job arrives at t=0, burst=4s, first runs at t=2, completes at t=6
→ Turnaround = 6–0 = 6s  |  Response = 2–0 = 2s  |  Waiting = 6–4 = 2s

⏳ Job Starvation

Starvation happens when a low-priority process never gets CPU time because higher-priority processes keep arriving. Common in priority scheduling and SJF (Shortest Job First).

  • Solution: Aging – gradually increase the priority of waiting processes
  • Linux CFS (Completely Fair Scheduler) avoids starvation using virtual runtime (vruntime)
$ nice -n 19 ./my_low_prio_task # run with lowest priority
$ renice -n -5 -p 1234 # increase priority of PID 1234

⏸️ Suspend State (SIGSTOP / kill -19)

A process receiving SIGSTOP enters the T (Stopped) state. It remains in memory, holding all its resources, but gets no CPU time. SIGCONT resumes it from exactly where it left off.

$ ./signals & # run your program in bg
$ kill -19 %1 # suspend (SIGSTOP)
$ ps aux | grep signals # STATUS column shows 'T'
$ kill -18 %1 # resume (SIGCONT)

🦘 setjmp / longjmp

Non-local goto — jump across function boundaries without unwinding the call stack normally. Often used as a C-level exception mechanism.

FunctionWhat it does
setjmp(env)Saves CPU state (registers, SP, PC) into env. Returns 0 on direct call
longjmp(env, val)Restores saved state; setjmp appears to return val (never 0)
#include <setjmp.h>

jmp_buf env;

void risky() {
    longjmp(env, 42);  // jump back!
}

int main() {
    int r = setjmp(env);   // save state
    if (r == 0) {
        risky();           // triggers jump
    } else {
        printf("Caught: %d\n", r);  // prints 42
    }
}
C
⚠️ Caveats
  • Local variables may be clobbered — declare them volatile
  • Does NOT call destructors or run cleanup code in skipped frames
  • For signal handlers use sigsetjmp/siglongjmp

🧵 Threads & Context Switching

Threads within the same process share code, heap, globals, and file descriptors. Only the stack and CPU registers are per-thread.

Why context switching is cheaper for threads

  • Process switch: kernel saves/restores all registers + flushes TLB (memory map) → expensive
  • Thread switch (same process): saves/restores only registers + stack pointer. No TLB flush needed → much cheaper
#include <pthread.h>

void *worker(void *arg) {
    printf("Thread %d running, PID=%d\n", *(int*)arg, getpid());
    return NULL;
}

int main() {
    pthread_t t;
    int id = 1;
    pthread_create(&t, NULL, worker, &id);
    pthread_join(t, NULL);  // wait for thread
}
// Compile: gcc -lpthread
C

⚙️ system() Function

system(cmd) runs a shell command synchronously — it blocks until the command finishes.

#include <stdlib.h>

int main() {
    system("ls -l /tmp");   // forks → exec /bin/sh -c "ls -l /tmp" → wait
    printf("ls done\n");    // only runs AFTER ls completes
}
C
💡 Under the hood

Internally: fork() → child calls execve("/bin/sh", ["-c", cmd]) → parent calls wait(). Because of wait(), execution is purely sequential.

📄 fopen – Buffered File I/O

fopen is a library call that wraps the open() system call, adding a user-space buffer (FILE*). Writes go to the buffer first; fclose/fflush flushes to kernel.

#include <stdio.h>

int main() {
    // Open for write (creates if not exists)
    FILE *f = fopen("/tmp/test.txt", "w");
    if (!f) { perror("fopen"); return 1; }

    fprintf(f, "Line 1\nLine 2\n");
    fclose(f);   // flush + close

    // Open for read
    f = fopen("/tmp/test.txt", "r");
    char buf[64];
    while (fgets(buf, sizeof(buf), f))
        printf("%s", buf);
    fclose(f);
}
C
ModeMeaning
"r"Read only (file must exist)
"w"Write only (create/truncate)
"a"Append (create if not exists)
"r+"Read + write (must exist)
"w+"Read + write (create/truncate)
"b"Binary mode (append to any above)

🍴 fork() – Creating Child Processes

fork() creates an exact copy of the calling process. After fork(), both parent and child execute concurrently (execution point: right after the fork call).

#include <unistd.h>
#include <stdio.h>
#include <sys/wait.h>

int main() {
    pid_t pid = fork();

    if (pid < 0) {
        perror("fork");     // error
    } else if (pid == 0) {
        // ── CHILD: fork returned 0 ──
        printf("[child]  PID=%d PPID=%d\n", getpid(), getppid());
    } else {
        // ── PARENT: fork returned child's PID ──
        printf("[parent] PID=%d child=%d\n", getpid(), pid);
        wait(NULL);         // reap child (prevent zombie)
    }
    return 0;
}
C
🧬 Copy-on-Write (COW)

After fork(), the child doesn't immediately get a full copy of memory. Instead, both share the same physical pages. A copy is made only when either process writes to a page — this makes fork() very fast.

🔌 Sockets – Network IPC

A socket is a file-descriptor to send/receive data over a network or between processes. TCP sockets are stream-based (reliable, ordered). UDP sockets are datagram-based (fast, unreliable).

/* ── TCP Server (bind, listen, accept) ── */
#include <stdio.h>
#include <string.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <unistd.h>

int main() {
    int srv = socket(AF_INET, SOCK_STREAM, 0);  // TCP socket

    struct sockaddr_in addr = {
        .sin_family = AF_INET,
        .sin_port   = htons(8080),     // port 8080
        .sin_addr.s_addr = INADDR_ANY  // all interfaces
    };

    bind(srv, (struct sockaddr *)&addr, sizeof(addr));
    listen(srv, 5);   // backlog 5 connections

    printf("Listening on :8080\n");
    int cli = accept(srv, NULL, NULL);      // blocks until client connects

    char buf[256];
    int n = read(cli, buf, sizeof(buf)-1);
    buf[n] = '\0';
    printf("Received: %s\n", buf);
    write(cli, "Hello from server!\n", 19);

    close(cli);
    close(srv);
    return 0;
}
C
$ gcc server.c -o server && ./server &
$ echo "Hi" | nc localhost 8080 # test with netcat
